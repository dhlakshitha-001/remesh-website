REMESH Resilient Mesh Sensor Hub

A self-healing LoRa mesh IoT network for real-time flood detection and community alerting in the Deduru Oya river basin, Sri Lanka — operating entirely without internet or cloud infrastructure.

Explore Research → View Documents
Best ML Recall 99.42% · FN = 3 of 1,096 test records · Logistic Regression
Abstract

REMESH is a resilient, self-healing LoRa-based mesh IoT early warning system designed for real-time flood detection and alerting in the Deduru Oya river basin — Sri Lanka's fifth largest river basin. During Cyclone Ditwah in November 2025, the basin experienced catastrophic flooding affecting over 340 Grama Niladhari Divisions. Existing monitoring systems failed because they depend on internet connectivity that is disrupted during major flood events. REMESH operates entirely without internet infrastructure: sensor data traverses a 433 MHz LoRa mesh with custom binary compression (~88% payload reduction) and XOR encryption, while five ML classifiers trained on 10 years of real hydrological data from the Moragaswewa gauging station (2015–2025) provide binary flood/no-flood inference. Logistic Regression achieved 99.42% recall with only 3 false negatives on the 1,096-record test set. Alerts are delivered simultaneously via SMS, dashboard, and on-node audio. Total prototype hardware cost: Rs. 34,740 (~$115 USD).

System Components
🌊

Distributed IoT Nodes

Three-tier hardware architecture deployed in watertight solar-powered enclosures. Main nodes carry 7 sensors: rain detection, water level, turbidity, pH, water flow, temperature, and humidity — plus GPS and INA219 battery monitoring. Repeater nodes extend LoRa coverage and serve an offline local dashboard. End nodes bridge the mesh to the internet and trigger audio alerts via DFPlayer Mini.

LoRa Routing, Encryption & Compression

When WiFi fails, the system automatically switches to LoRa. A custom binary SensorPacket struct compresses telemetry from ~600 bytes JSON to ~70 bytes (~88% reduction), fitting within LoRa's 255-byte payload limit. XOR rolling-counter encryption secures each packet. Multi-hop routing uses loop prevention (hop-trail check), duplicate suppression (15-entry timestamp cache), and randomised 1.5–6 s forwarding delay for collision avoidance.

ML Flood Prediction

Five classifiers evaluated on a real 10-year dataset (2015–2025) from the Moragaswewa gauging station, supplemented by Deduru Oya basin rainfall and Open-Meteo meteorological data (4,383 daily records, 11 features). SMOTE oversampling resolved the 1.8% flood prevalence. Logistic Regression was selected — highest recall (99.42%), fewest false negatives (FN = 3), and lowest ESP32 inference overhead. Risk mapped to four levels: Low / Moderate / High / Critical.

Dashboard & Management

Centralised Laravel web application ingesting telemetry from IoT nodes via REST API. Features: real-time geographical node visualisation, live Rivernet.lk river level integration, ML flood risk display, SMS alerts to predefined groups, on-node DFPlayer audio alert triggering, geo-tagged citizen emergency reporting visualised as a heat map, district/zone-based monitoring, and role-based access control.

Deployment Context
Deduru Oya River Basin, Sri Lanka

The fifth largest river basin in Sri Lanka. During Cyclone Ditwah (November 2025), the Deduru Oya Reservoir released water at approximately 16,200 ft³/s around November 26–27, triggering widespread flooding across Puttalam and Kurunegala Districts. The absence of localised, offline-capable early warning amplified casualties — the core problem REMESH is designed to solve.
